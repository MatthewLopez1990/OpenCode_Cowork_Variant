## Format

Structure the audit checklist as follows:

1. **AUDIT OVERVIEW**
   - Audit title and scope
   - Audit period
   - Audit type (financial, operational, compliance, IT, internal controls)
   - Audited entity / department / process
   - Audit objectives
   - Applicable standards and frameworks (GAAS, GAAP, IFRS, SOX, COSO, COBIT, etc.)

2. **PLANNING CHECKLIST**

   | # | Task | Status | Assigned To | Due Date | Notes |
   |---|------|--------|------------|----------|-------|
   | 1 | Define audit scope and objectives | | | | |
   | 2 | Identify applicable standards and regulations | | | | |
   | 3 | Assess inherent and control risk | | | | |
   | 4 | Determine materiality thresholds | | | | |
   | 5 | Review prior audit findings and follow-up items | | | | |
   | 6 | Prepare audit program and procedures | | | | |
   | 7 | Send engagement letter / notification | | | | |
   | 8 | Schedule entrance conference | | | | |
   | 9 | Request preliminary documents and data | | | | |
   | 10 | Assign audit team and responsibilities | | | | |

3. **INTERNAL CONTROLS ASSESSMENT**

   For each control area, assess using the COSO framework components:

   **Control Environment**
   - [ ] Organizational structure and reporting lines reviewed
   - [ ] Code of conduct and ethics policies reviewed
   - [ ] Management tone and commitment to integrity assessed
   - [ ] Competency standards and training programs evaluated

   **Risk Assessment**
   - [ ] Risk identification process reviewed
   - [ ] Risk mitigation strategies evaluated
   - [ ] Change management procedures assessed
   - [ ] Fraud risk assessment reviewed

   **Control Activities**
   - [ ] Authorization and approval procedures tested
   - [ ] Segregation of duties verified
   - [ ] Physical controls over assets examined
   - [ ] Reconciliation procedures tested
   - [ ] IT general controls reviewed (access, change management, operations)

   **Information and Communication**
   - [ ] Financial reporting processes reviewed
   - [ ] Information systems and data integrity assessed
   - [ ] Internal communication channels evaluated
   - [ ] External reporting requirements verified

   **Monitoring Activities**
   - [ ] Ongoing monitoring processes reviewed
   - [ ] Internal audit function assessed
   - [ ] Deficiency identification and remediation process evaluated
   - [ ] Management reporting and escalation procedures reviewed

4. **SUBSTANTIVE TESTING PROCEDURES**

   Organize by financial statement area or process:

   **Revenue and Receivables**
   - [ ] Revenue recognition policy reviewed for compliance with ASC 606 / IFRS 15
   - [ ] Sample of revenue transactions tested for proper recognition
   - [ ] Accounts receivable confirmations sent and reconciled
   - [ ] Allowance for doubtful accounts methodology reviewed
   - [ ] Cutoff testing performed

   **Expenses and Payables**
   - [ ] Expense classification and authorization reviewed
   - [ ] Sample of disbursements tested against supporting documentation
   - [ ] Accounts payable completeness tested (search for unrecorded liabilities)
   - [ ] Accrued liabilities reviewed for reasonableness
   - [ ] Related party transactions identified and reviewed

   **Payroll**
   - [ ] Payroll registers reviewed and sampled
   - [ ] Employee existence verified
   - [ ] Pay rates authorized and documented
   - [ ] Payroll tax compliance verified
   - [ ] Benefits and deductions reviewed

   **Cash and Investments**
   - [ ] Bank reconciliations reviewed and tested
   - [ ] Bank confirmations obtained
   - [ ] Investment existence and valuation verified
   - [ ] Cash restrictions or covenants reviewed

   **Fixed Assets**
   - [ ] Asset register reviewed for completeness
   - [ ] Additions and disposals tested
   - [ ] Depreciation calculations verified
   - [ ] Impairment assessment reviewed
   - [ ] Physical verification performed (sample)

   **Inventory** (if applicable)
   - [ ] Inventory count procedures observed
   - [ ] Valuation methodology reviewed (FIFO, LIFO, weighted average)
   - [ ] Obsolescence reserves assessed
   - [ ] Cutoff procedures tested

5. **COMPLIANCE TESTING**
   - [ ] Applicable laws and regulations identified
   - [ ] Compliance monitoring procedures reviewed
   - [ ] Licenses and permits verified as current
   - [ ] Regulatory filings reviewed for timeliness and accuracy
   - [ ] Data privacy and security compliance assessed
   - [ ] Tax compliance reviewed

6. **FINDINGS TEMPLATE**

   For each finding, document:

   | Field | Description |
   |-------|-------------|
   | Finding # | Sequential identifier |
   | Title | Brief descriptive title |
   | Condition | What was found (factual description) |
   | Criteria | What should be (standard, policy, regulation) |
   | Cause | Why it occurred (root cause analysis) |
   | Effect | Impact or potential impact (quantify if possible) |
   | Risk Rating | Critical / High / Medium / Low |
   | Recommendation | Specific corrective action |
   | Management Response | (To be completed by management) |
   | Responsible Party | (To be completed by management) |
   | Target Date | (To be completed by management) |

7. **WRAP-UP CHECKLIST**
   - [ ] All procedures completed and documented
   - [ ] Findings reviewed with management
   - [ ] Management responses obtained
   - [ ] Draft report prepared and reviewed
   - [ ] Exit conference scheduled and conducted
   - [ ] Final report issued
   - [ ] Working papers organized and archived
   - [ ] Follow-up schedule established

## Formatting Guidelines

- Use checkboxes ([ ]) for all checklist items.
- Use tables for structured data and findings.
- Number all items sequentially within sections.
- Include status columns: Not Started / In Progress / Complete / N/A.
- Cross-reference findings to specific procedures and controls.
- Use consistent risk rating definitions throughout.
